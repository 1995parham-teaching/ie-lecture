IaaS

Infrastructure as a service (IaaS) refers to online services that abstract the user from the details of infrastructure like physical computing resources, location, data partitioning, scaling, security, backup etc.

    PaaS

    Platform as a service (PaaS) or application platform as a service (aPaaS) is a category of cloud computing services that provides a platform allowing customers to develop, run, and manage applications without the complexity of building and maintaining the infrastructure typically associated with developing and launching an app.

    Types of Virtualization

    • Partitioning: This is a type of virtualization in which a CPU is devided into different parts, and each part works as an individaul system.
    • Full virtualization: In full virtualization, a virtual machine is used to simulate regular hardware while not being aware of the fact that is's virtualized.
      • Software-based: Uses binary translation to virtualize the execution of sensitive instruction sets while emulating hardware using software, which increase overhead and impacts scalibility.
      • Hardware-based: Removes binary translation from the equation while interfacing with a CPU's virtualization features (AMD-V, Intel VT), which, in turn, means that instruction sets are being executed directly on the host CPU
    • Paravirtualization: This is a type in which the guest OS understands the fact that it's being virtualized and needs to be modified, along with its drivers, so that it can run on top of the virtulization solution.
    • Hybrid virtualization: This is a type of virtualization that uses full virtualization and paravirtualization's biggest virtues.
    • Container-based virtualization: This is a type of application virtualization that uses containers. A container is an object that packages an application and all its dependencies so that the application can be scaled out and rapidly deployed without needing a virtual machine or a hypervisor.

    Containers

    Containers are an abstraction at the app layer that packages code and dependencies together. Multiple containers can run on the same machine and share the OS kernel with other containers, each running as isolated processes in user space.
